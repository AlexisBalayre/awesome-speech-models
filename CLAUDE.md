# awesome-speech-models

A curated knowledge hub (an "awesome list") of open-source tools, models, datasets, and research for **speech AI**. The deliverable is accurate, well-organized Markdown — there is no application code here.

## Structure
- `README.md` — top-level hub and index.
- `asr/`, `tts/`, `speech-to-speech/` — one domain per folder. Every domain README uses the **same sections**: Tools & Libraries · Models · Datasets · Benchmarks & Leaderboards · Optimization & Inference · Finetuning & RL · Papers & Research · Learning.
- `CONTRIBUTING.md` — the canonical entry format and curation rules. **Read it before editing.**

## Adding or editing entries
- Use the exact format from `CONTRIBUTING.md`: `- **[Name](url)** — one line. ` plus backtick-tags.
- **Verify every link is live and points to the right project before adding it.** Use WebFetch/WebSearch — never guess or invent a URL or description.
- File it under the correct **domain** and **section**. Grep the repo first to avoid duplicates.
- Open source preferred; prefer actively maintained projects. One line per entry.
- Within a section, order roughly by relevance/popularity.

## Don't
- Don't pad sections with low-quality or unmaintained entries — quality over quantity.
- Don't fabricate links, names, or descriptions. If unsure, verify it or leave a `<!-- TODO -->`.
- Don't reformat or reorder unrelated entries while making a small change.
